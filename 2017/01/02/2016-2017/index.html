2016->2017

发表于 2017-01-02 分类于 生活感想

2017年了，今天把以前的文章看了下，不禁笑了笑，感觉自己还是蛮逗逼的，尽吹牛逼、放卫星，难怪别人叫我逼哥，哈哈哈，不过高中的外号，大学断了，竟然又在工作中出现了，硬却失挺。今天把 hexo 又折腾了下，换了主题even，感谢这位大三的作者(大三😯)，码字的激情又上来了。把评论由多说更换成了disqus，所以很有可能需要FQ才能看到相关评论，我猜想评论的人应该比较少吧。

#2016
2016年，总的来说还是挺失望的，因为这一年，我真的有努力的花时间去学习过，但是效果却不明显。

##关于加班
2016年元旦那天搞到凌晨4点多才下班，尽管说是要上线，所以就作死的赶着上线，其实当时貌似自己心里也没啥底，呵呵。昨天看了纯银的去你妈的996，我不记得是哪个大V以前在微博上说过，”一切的加班都是领导的无能”，原话是不是这样的，我记得不是很清楚了，反正大致意思就是这样。个人非常赞同纯银的观点，合理常态加班。其实，对加班的态度可以看出一个员工对工作、上司领导、产品、公司前途等的满意度。如果经常加班，又特么没搞出什么成绩来，那员工的激情肯定会下降（个人觉得激情是非常重要的东西）。如果没事做，也要加班的话，会导致工作效率降低，反正要加班的，那我还不如留到下班后弄啊，这样对员工自己的发展是有弊端的，因为你会养成这样的习惯。一个习惯坏习惯的养成很容易，而一个好习惯的养成…呵呵。2016年加班貌似跟2015年差不多，然而所属的项目也没作出啥成绩了。2016年底被分到另外一个项目组，听项目管理的描述，其实对整个产品还是充满憧憬，希望能够干出一点成绩来。所以2016年底，我都会加班做一下项目底层的事情，可以把它理解为”架构”咯，其实，自己有几斤几两，自己心里还是清楚的。作为iOS端的负责人，心里非常想把这个产品做好的（以后出去吹牛都底一点不，哈哈哈哈），而困难的事则是怎么让团队的同事同样有激情，我觉得用行动感染最合适。

##关于离职
在公司待了快两年了，我们部门离职率很低的，今年身边经常沟通合作的同事走了3个（酿哥，zzp表哥，尹哥），都是回老家发展了。前段时间，跟尹哥聊天，得知在长沙现在他的工作状态，很惊讶！竟然晚上加班还可以调休，周末加半天就可以调休一天，WTF？！！！现在的我肯定不愿过这样的生活的，我还想好好折腾下自己。

##关于压力
今年，经常看到一些创业公司死了，员工讨薪等新闻。iOS目前就业情景非常不好，貌似现在培训结构都不培训iOS，都大量培训前端了。我还经常担心，如果自己特么某天被裁了，该怎么玩？一些比较重要的底层东西不是很懂，swift也没实践过，react native、weex也只是听过，js也不会。。。。。。感觉最近自己危机感比较严重。而唯一的途径出路，都心里有底哈，问题就看自己的实践了。

##关于提升
今年博客写的比较少，大部分是自己工作中遇到的坑，纵向、横向也没啥扩展。认真看过几个开源项目了，提了几个issue、pr，完成了工作中几个技术难点。Apple document 也有看几篇，但是没有全部看懂，笔记做的比较low。没写过轮子。总体来说，2016年相对于2015年有所提升，但效果不是很明显。值得高兴的是，年底的时候貌似隐隐约约找到了一种比较适合自己的学习方法（还没有大量实践）。

#2017
2017年是毕业后的第三年，人生第三个轮回的开始。工作后的这两年，工作能力没有得到大幅度的提升。事不过三，希望2017年，能够运用适合自己的学习方法提升效率。能够在非常重要的时间段看到耗子叔的这篇文章技术人员的发展之路，非常感谢！展望一下，大概有下面几个方面吧。

##提升效率
昨天老妹看我在刷技术相关的微博，就跟我开玩笑说，就知道每天搞学习，又没啥成长，还好在你们公司你是算好的，不然就……哈哈哈哈哈哈。当时我无语以对。对于我来说，如何提升效率就很重要了，不然吹牛逼说在学习，结果还是一坨屎，那就尴尬了。怎么样在有效的时间里提升自己，在其他的时间里，能够好好的玩，去了解一下其他方法的知识。上班前让自己提前进入工作状态很重要，可以看看peak君的一个让脑子快速进入编程状态的小Tip。

##做好笔记
我记得8月份左右那段时间，下班时间、周末时间，都会看 The Swift Programming Language (Swift 3)、View Programming Guide for iOS等苹果官方文档，笔记有做，但是感觉都是笔记肤浅的比较，并且看完一篇就完了，没有第二次深入做笔记。在工作中，遇到的问题，其解决办法以及相关的知识点，都要做好笔记，最好是能够描述下上下文，尽量避免以后再踩类似的坑。

##实践练习
上面提到有看Apple document，但是没有运用到实践中，增加理解（尽管apple都会有相关的代码示例）。主动学习一个知识点以后，应该要考虑到实际情况中。看废土的博文，他做的非常好，貌似一些大牛都是这样子的，只不过最近看了他的博文，印象比较深刻。

##纵向深挖
泛泛知道几种东西和深入了解掌握一种东西，这其他的区别都知道的。看 Apple document 其实也算是吧，Apple document 里面说的很清楚，它会介绍由来、原因等，但同时有非常广泛，这个点同时又牵扯到其他的知识点，到这里，你就得自己好好把握了（在纵向和横向要有个权衡）。当然，如果你时间够，你也可以把整个知识体系都了解完的。

##横向扩展
专业技能需要纵向深挖，但是其他的还是得大概知道是怎么回事的。

##注意身体
身体是革命的本钱，以后争取每周跑步 2-3 次，不然就浪费了深圳这么好的环境。

##干干干干
说了这么多，最重要的就是干！作死的干！！希望能够完成心里的几个小目标。争取明年写总结的时候，不会像今天这么无赖。
